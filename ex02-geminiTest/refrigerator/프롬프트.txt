당신은 뛰어난 프론트엔드 개발자이자 UI/UX 디자이너입니다.
냉장고에 남은 식재료를 활용해 레시피를 추천해 주는 '냉장고 비우기' 웹 앱의 프로토타입을 HTML, Tailwind CSS, 바닐라 JavaScript를 사용하여 하나의 파일(prototype.html)로 만들어주세요.

[핵심 목표 및 컨셉]
1. 사용자가 냉장고 사진을 업로드하거나 샘플 이미지를 선택하면, AI가 식재료를 인식(시뮬레이션)합니다.
2. 인식된 식재료 목록을 관리할 수 있는 '냉장고 보관함' 기능을 제공합니다.
3. 보관함의 재료를 바탕으로 로컬 데이터베이스의 레시피와 매칭하여 추천 목록을 보여줍니다.
4. Gemini API를 활용하여 현재 재료에 딱 맞는 'AI 특제 시그니처 요리'를 동적으로 생성합니다.
5. 전체적인 디자인은 모던하고 깔끔하며, 에메랄드(Emerald)와 슬레이트(Slate) 색상을 주조색으로 사용하여 신선한 느낌을 주어야 합니다.

[주요 UI/UX 구성 요소]
- 네비게이션 바: 로고 아이콘, 앱 이름, 'AI Live 연결됨' 상태 표시기
- 히어로 섹션: 앱의 목적을 알리는 매력적인 카피와 그라데이션 배경
- 메인 레이아웃: 12컬럼 그리드를 사용, 왼쪽(5컬럼)은 사진 업로드 및 재료 관리, 오른쪽(7컬럼)은 레시피 추천 및 AI 요리 제안 배치

[상세 기능 요구사항]

1. 이미지 입력 및 분석 (왼쪽 영역 상단)
- 직접 업로드와 샘플 이미지 선택 탭 구성
- 직접 업로드 시 파일 첨부 UI 제공 및 썸네일 미리보기 표시
- 샘플 선택 시 3가지 옵션(다이어터, 국/찌개, 고기 러버) 제공
- '스마트 식재료 분석 시작' 버튼 클릭 시, 업로드된 이미지는 Gemini 2.5 Flash API (Vision 기능)를 호출하여 식재료를 추출하도록 구현
- API 호출 시 로딩 스피너 및 진행 상태 텍스트 애니메이션 표시
- 샘플 이미지는 미리 정의된 식재료 목록을 사용(시뮬레이션)
- 분석 실패 시 비상 폴백(임시 식재료 주입) 로직 구현

2. 인벤토리 매니저 (왼쪽 영역 하단)
- 인식된 식재료가 칩(Chip) 형태로 표시되며, 삭제 버튼 제공
- 식재료 수동 추가를 위한 텍스트 입력 폼 구현
- '전체 비우기' 버튼 제공

3. 레시피 매칭 리스트 (오른쪽 영역 상단)
- 로컬 레시피 데이터베이스(최소 5개 이상의 구체적인 레시피: 계란말이, 김치찌개, 볶음밥 등) 구축
- 사용자의 인벤토리 재료와 레시피의 필수 재료를 비교하여 '매칭률(%)'을 계산하고 프로그레스 바 형태로 시각화
- '조미료는 보유로 가정' 체크박스를 통해 매칭률 계산 로직 동적 변경 지원
- 일부 재료(예: 대파->쪽파, 햄->소시지)는 대체 가능하도록 매핑 로직 포함
- 매칭률이 높은 순으로 정렬하여 표시
- 각 레시피의 '조리법 보기' 버튼 클릭 시 모달 창 오픈

4. 상세 레시피 모달 창
- 모달 창 내부에 요리 정보(시간, 난이도, 칼로리) 표시
- 필요 식재료 체크리스트 (보유/부족 상태 배지 표시)
- 조리 단계별 가이드라인 (체크박스 형태로 완료 표시 가능)
- 부족한 재료를 네이버 쇼핑 등으로 검색하는 가상 연동 버튼 구현

5. AI 특제 요리 제안 (오른쪽 영역 하단)
- 'AI 셰프 호출' 버튼 클릭 시 현재 인벤토리 재료 목록을 조합한 프롬프트 생성
- Gemini 2.5 Flash API (Text 기능)를 호출하여 창의적인 퓨전 요리 레시피 1종 생성
- API 호출 중 로딩 UI 표시
- 반환된 HTML 형식의 레시피를 화면에 렌더링
- API 호출 실패 시 로컬에 저장된 폴백 레시피 출력

[기술적 요구사항]
- Tailwind CSS CDN을 사용하여 스타일링
- Lucide Icons (CDN)를 사용하여 아이콘 적용
- 폰트는 Noto Sans KR 적용
- 브라우저 알림 대신 커스텀 Toast 알림 시스템 구현
- API 호출 시 지수 백오프(Exponential Backoff) 적용
- 모든 코드는 하나의 html 파일 안에 포함될 것 (CSS는 Tailwind 클래스 및 <style> 태그, JS는 <script> 태그 사용)
